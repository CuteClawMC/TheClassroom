# Classroom Subnet

## Raise And Economics Draft

Date: 2026-03-21  
Prepared by: DC

## Purpose

This document is a proposed economics page for a Bitstarter-style raise. It is intentionally conservative.

The goal is not to maximize headline excitement. The goal is to give the proposal a credible financing shape that matches the current state of the project:

- real prototype
- real technical ambition
- unresolved validator design work
- a clear need for build capital

## Current Bitstarter framing to mirror

Visible current Bitstarter subnet pages use a structure like:

- raise target in TAO
- alpha allocation amount
- discount tiers
- lock-up period
- brief use-of-funds note

Reference examples:
- [Bitstarter main site](https://www.bitstarter.ai/)
- [Quasar subnet page](https://app.bitstarter.ai/subnets/quasar/)

That means Classroom should show clear terms rather than vague economics language.

## Recommended classroom raise posture

My recommendation is to position Classroom as a protocol-build raise, not an emissions-theater raise.

That means:

- no exaggerated APY promises
- no “print money” framing
- no buyback theater as the main story
- no reliance on speculative performance claims from the trading side

The raise should fund:

- validator integrity architecture
- subnet engineering
- data infrastructure
- miner simulation and onboarding
- initial Oracle benchmark work

## Proposed terms

### Headline structure

- `Soft target`: `300T`
- `Full allocation reserve`: `125,000 alpha`
- `Reference alpha price`: `0.0040 TAO per alpha`
- `3-month lock-up tier`: `25% discount`
- `6-month lock-up tier`: `40% discount`

### Implied pricing

- reference price: `0.0040 TAO`
- `25%` discount price: `0.0030 TAO`
- `40%` discount price: `0.0024 TAO`

### Why this structure works

With `125,000 alpha` reserved:

- if the full reserve clears at the `6-month / 40%` tier, the raise lands at `300T`
- if the full reserve clears at the `3-month / 25%` tier, the raise lands at `375T`

That gives Classroom a clean financing story:

- `300T` is the minimum fully funded operating target
- `375T` is the upside case if shorter lock-ups dominate

This is cleaner than pretending the exact tier mix is already known.

## Suggested public copy

`Classroom seeks a 300T operating raise through a reserved allocation of 125,000 alpha. Contributors can choose between a 25% discount with a 3-month lock-up or a 40% discount with a 6-month lock-up. The raise is designed to fund validator integrity, protocol engineering, and the data infrastructure required to turn the current prototype into a subnet-grade scientific process.`

## Why 300T is the right soft target

A 300T soft target is large enough to look serious and small enough to stay credible for a prototype-to-subnet transition.

It implies the raise is funding:

- engineering and protocol build-out
- not indefinite narrative runway
- not a speculative treasury game

It also fits the stage of the project better than trying to raise as if the subnet is already fully de-risked.

## Suggested use of funds

### Allocation by bucket

- `30%` protocol engineering and subnet implementation
- `20%` validator integrity and anti-gaming research
- `15%` data infrastructure and storage
- `10%` miner simulation, testnet operations, and onboarding
- `10%` model training and benchmark evaluation
- `10%` core operations and founder runway
- `5%` security review, monitoring, and contingency

### Translation into actual message

The message to investors should be:

- most capital goes to making the subnet real and defensible
- a smaller slice goes to turning the resulting database into benchmark model value
- very little goes to vanity overhead

## Treasury guardrails

This part matters because Classroom touches markets.

The treasury policy should be cleaner than the average “AI x finance” raise.

### Recommended guardrails

1. Raised TAO funds protocol build, not discretionary trading.
2. No performance promises based on prototype trading PnL.
3. Treasury use should be milestone-based, not founder-discretion based.
4. Reserve a modest contingency buffer instead of promising aggressive buybacks.
5. If buyback or emissions-support logic is added later, it should be documented separately and not used as the core raise hook.

This makes the raise more credible.

## Milestone gating for spend

The easiest way to make the economics believable is to tie spending to protocol milestones.

### Suggested gates

#### Gate 1: validator testnet and miner harness

Unlock priority spending for:

- protocol engineering
- simulation harnesses
- initial infra

#### Gate 2: first accepted payload stream

Unlock priority spending for:

- data infra scaling
- miner onboarding
- validator monitoring

#### Gate 3: stable integrity scoring and rejection path

Unlock priority spending for:

- benchmark model training
- early Oracle-candidate work
- broader subnet preparation

This keeps the capital story disciplined.

## What I would not promise yet

I would explicitly avoid overcommitting on:

- fixed APY language
- hard buyback commitments
- guaranteed Oracle launch dates
- claims that alpha value will be directly tied to trading performance in the near term

Those promises would make the page weaker, not stronger.

## Bottom line

The strongest Classroom economics page is not the most aggressive one.

It is the one that makes a reviewer think:

- the founder understands what actually needs to be built
- the raise size matches the stage
- the terms are legible
- the treasury will be used to build protocol credibility, not theater
